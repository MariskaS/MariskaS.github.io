Code Style

Implementing Code Style for Frontend Teams

WHY

Improve and systematize work processes on projects.

  • Easy switching of developers between projects;
  • Facilitating the process of new developers entering projects;
  • Will speed up the process of creating new projects;

These are just some of the main key points.

HOW

  • Creating an agreement on a uniform Code Style;
  • Implementation of workflows to control the maintenance of Code Style;

Example of general workflow.

Workflows to control the maintenance of Code Style

  1. Strict adherence to Agile rules;
  2. Every pull request must go through Code Review (The group of people specified in the project documentation in confluence);
  3. Merge to main branch only after successfully passing .eslint & .csslint;
